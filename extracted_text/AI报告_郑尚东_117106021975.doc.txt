1、研究的主要内容和意义
　　随着 Internet 的快速发展，Web 上的信息与资源日益膨胀。面对海量的信息资源，如何更快更好的获取需要的资源成为人们日益关注的问题。通用搜索引擎返回的结果页面中含有大量的"噪声"页面，需要人为的去挑选自己所关注的主题。各类搜索引擎：垂直搜索引擎，电子商务搜索引擎，分布式搜索引擎，可视化搜索引擎等等等等，为了给人们提供了更快，更专业，更精准的网络资源的检索服务，都应运而生。
　　所以智能搜索的研究，在信息日益膨胀的今天就显得尤为重要，不管是对企业、个人、教育机构和行政机关，智能搜索的研究的重要性都展现在了方方面面。虽然说目前这些搜索引擎（包括新一代搜索引擎）仍然存在着高覆盖率（Recall）和低精确率（Precision）的问题，这也就意味着这些搜索引擎，依旧无法提供准确有效地满足用户需求的信息服务但是我们相信，随着互联网信息资源的不断增长，研究智能搜索技术主动地为满足某一类信息需求的用户提供相应的信息服务！
　　
　　
2、国内外研究现状和发展水平
　　搜索引擎（Search Engines）是指在WWW中能够主动搜索信息、组织信息并能提供查询服务的一种信息服务系统。搜索引擎主要通过网络搜索软件或网站登录方式将WWW上大量网站的页面信息收集到本地,经过加工处理后建成数据库，从而能够对用户提出的各种查询请求作出响应，提供用户所需要的信息地址。搜索引擎提供的导航服务已经成为互联网上非常重要的网络服务，因此搜索引擎也被称为"网络门户"。
2.1国内外研究现状
　　近年来，人们在利用着互联网的高速发展带来的海量信息的同时，也遭受"信息迷航"和"信息过载"的困扰。在这样的背景下，对垂直搜索引擎技术的研究和应用已显得非常迫切，搜索引擎技术的研究热点在逐渐由传统搜索引擎向垂直搜索引擎转变。目前，国内外的很多研究机构和 IT 企业都投入了大量的资金和人力从事垂直搜索引擎的相关研究工作。具有代表性的面向特定主题的垂直搜索引擎系统主要有以下一些应用。 
　　国外： 
　　①Berkeley 大学的 Focuse Project 这个系统是由一个印度裔的科学家 S.Charkrabarti 带头从事开发。该系统通过两个程序来指导主题网络蜘蛛，一个是主题分类器 Classifier，用来计算下载的文档与预定主题的相关程度；另一个则是净化器 Distiller，用来确定哪些链接指向相关资源的页面。 
　　②美国国家科学数字图书馆的 Collection Building Program(CBP)。
　　这个项目旨在为科学、数学、工程和技术创建大规模的在线数字图书馆，试图研究在某一个主题上资源自动建设的可能性。CBP 的特点是：由于其只面向教育和教学，因此主题精确度高；不存储资源原文，只提供 URL；只需要少量输入，系统就可以全自动将有关该主体的最相关的有限数量 URL 返回给用户。
　　国内： 
　　①奇虎 www.qihoo.com。号称中文论坛第一门户，以收集整理 BBS 存储着的大量信息为主，其独创的 People Rank 技术主要用来抓取和识别论坛和博客中的信息，并按照利于人们阅读的方式予以呈现。 
　　②酷讯 www.kuxun.cn。与百度、谷歌的网页搜索引擎搜索不同，酷讯是一款以即时的生活信息为检索对象的专业搜索引擎。主要通过整理编辑各类票价、酒店等信息，为用户提供租买房、买火车票等服务。虽然垂直搜索引擎在国内外都已成为互联网研究的热点，并得到了一定的应用。但现在各类垂直搜索引擎普遍还处在探索和发展阶段，上述几种垂直搜索引擎都或多或少的采用了人工整理分类策略。同时，当前垂直搜索引擎都面临着一个亟需解决的"主题漂移"难题。所谓"主题漂移"就是指随着网页的不断下载，网页数据库中的主题信息淹没在其他非主题信息之中。为了能有效解决"主题漂移"问题，很多学者对此进行了研究， Cho 和 Hersovici 等人提出的 Best First Search  算法构建一个 URL 链接列表，然后按照某种评价选择策略选择出最好的 URL 进行访问；De Bra 等提出的 Fish 算法用网页内容文本主题相关性的判别来预测链接网页的优先级，相关于某一主题链接到的页面趋向于拥有同一题，应该优先下载；M.Hersovici 等人提出的 shark 算法认为每个链接的说明文字对此链接所指向的页面主题有着较高的预测能力，用来判别相关链接的主题相似度有很高的准确性。这些算法和思想对解决"主题漂移"问题具有一定的价值和借鉴意义。
2.2、搜索引擎的历史
　　现代意义上的搜索引擎鼻祖是1990年由蒙特利尔的麦吉尔大学学生Alan Emtage、Peter Deutsch、Bill Wheelan发明的Archie。当时World Wide Web还未出现，Archie是第一个自动搜索互联网上FTP网站文件的程序。Archie是一个可搜索FTP文件名的列表，用户必须输入精确的文件名搜索，然后Archie会告诉用户哪一个FTP地址可以下载该文件。Archie还不是真正意义上的搜索引擎。
　　随着Internet的快速发展，出现了专门用于检索信息的Robot程序，该程序能自动地不断重复地在网络上检索信息，这种行为很像一只蜘蛛在Internet上爬来爬去，所以又称为Spider程序。
　　1993年，互联网上出现了最早的Web浏览器Mosaic，次年Netscape推出了Navigator。浏览器的发展促使Web得到迅速推广，站点数目以惊人的速度增加。随着互联网的迅猛发展，信息按几何式增长，搜索引擎开始快速发展起来。
　　1994年7月，世界上出现了最早的真正意义上的搜索引擎--Lycos。除了相关性排序外，Lycos还提供了前缀匹配和字符相近限制，Lycos第一个在搜索结果中使用了网页自动摘要，而最大的优势还是它远胜过其它搜索引擎的数据量。同年4月，美国斯坦福大学的两位博士生David Filo和美籍华人杨致远共同创建了Yahoo！，并成功地使搜索引擎的概念深入人心，从此搜索引擎进入了快速发展时期。
　　一般把这一时期的搜索引擎被称为第一代搜索引擎。第一代搜索引擎初期还分为目录式搜索和全文式搜索两类。Yahoo!是第一代搜索引擎的代表，属于典型的目录式分类结构，而Alta Vista则属于全文式搜索引擎。第一代搜索引擎是以人工方式或半自动方式搜集、整理信息，往往以反馈结果的数量来衡量检索结果的好坏。而研究表明，第一代搜索引擎检索结果的相关性较差，资料更新缓慢。
　　随着网络信息的日益增长，以人工分类整理的搜索引擎已经不能满足用户搜索信息的要求。
　　自1998年开始，出现了一个搜索引擎空前繁荣的时期。随着Google的出现，第二代搜索引擎诞生。它们不仅拓展了搜索引擎的生存空间，而且还极大的提高了搜索的质量和效率。除Google外，Yahoo!在2004年也推出了自己的第二代搜索引擎--"一搜"（Yiso）。在国内，百度逐渐取代新浪、搜狐，成为中文第二代搜索引擎的老大。第二代搜索引擎中最具代表性、最成功的还是Google。总的来说，第二代搜索引擎依靠机器抓取，建立在超链分析基础之上，提高了查准率、查全率和检索速度。
　　2003年8月20日，中国搜索CEO陈沛首次提出了第三代搜索引擎的概念。2004年12月，中国搜索推出的网络猪3. 0被陈沛看成是第三代搜索引擎个性化信息门户的开始。
　　2006年4月18日，中国搜索发布了客户端搜索软件IG（Internet Gateway），而IG是网络猪的升级产品。另外，2004年8月，搜狐CEO张朝阳也提出了第三代互动搜索概念。
　　2005年9月，美国政府提出开始研制第三代搜索引擎。2005年10月，微软公布了该公司做第三代搜索引擎的构想。2007年4月，中国Jwmguagua（精武门呱呱）率先提出了第三代直潜全能搜索引擎技术系统。经过5 年多的探索和市场"历练"之后，全球第三代搜索引擎呈现出两个发展方向：一个是以Google为主的"技术驱动型"理念；一个是以Jwmguagua（精武门呱呱为主的"服务驱动型"理念。第三代搜索引擎是智能搜索引擎，呈现出多元化、智能化、个性化的发展趋向。
2.3、传统搜索引擎的缺陷
　　传统的搜索引擎技术一般有两种。
　　一种是使用网站分类技术，即把网站进行树状的归类，并对每个站点都有简略的描述。其优点是导航明晰，但成本高，详细信息丢失严重。
　　另一种是使用全文检索技术，全文检索技术处理的对象是文本，它能够对大量文档建立由字(词)到文档的倒排索引，在此基础上，用户使用关键词来对文挡进行检索时，系统将给用户返回含该关键词的数据。其解决了细节搜索问题，但是这又产生了新的缺陷，即返回的信息太多，另外，它也不能解决两个关键问题，即词汇的"忠实表达"和"表达差异"问题。从上面的分析可以看出,传统搜索引擎所使用的技术都难以解决用户"找信息难"的问题。造成这种困难的实质在于搜索引擎缺乏知识处理能力和理解能力，对要检索的信息仅仅采用机械的关键词匹配来实现。把信息检索从目前基于关键词层面提高到基于知识(或概念)层面，是解决问题的根本和关键，即将搜索引擎智能化，这就是我们所讲的智能搜索引擎，这是未来搜索引擎发展的趋势。
　　
2.4、搜索引擎的发展方向
　　1）向智能化方向发展
　　智能化是网络信息检索未来的主要发展方向。随着网络用户对检索精确度和效率的不断提高，越来越多的搜索引擎重视自身在检索功能和检索服务上的智能化程度，将涌现出智能搜索引擎。智能搜索引擎可以通过自然语言与用户交互，最大限度地了解用户的需求，它能为用户提供了一个真正智能化的、个性化的信息过滤和推送服务。因此，智能搜索引擎成为今后搜索引擎的发展趋势。
　　2）向个性化、特色化方向发展
　　个性化是指各网络检索工具注重内容的特色化和服务的个性化。个性化的核心是跟踪与分析用户的搜索行为，充分利用这些信息来提高用户的检索效率，并为用户提供全程帮助和服务，使得个性化搜索更符合每个用户的需求。
　　3）向多媒体化方向发展
　　未来的搜索引擎将在技术上更加成熟，在搜索内容上向多媒体化发展。视频、音频等多媒体信息的检索数据量高速增长，多媒体搜索引擎的发展迫在眉睫。同时，搜索引擎在向用户提供检索结果时仍可以向多媒体方向发展，使得检索结果形式多样化、生动化，更好地满足用户的需求。向多媒体化方向发展是搜索引擎发展的又一个新趋势。
　　4）向功能多样化方向发展
　　首先表现在可以检索的信息形态多样化，如文本、声音、图像、动画；其次表现在搜索引擎已不仅仅是单纯的检索工具，正在向其他服务范畴扩展，提供站点评论、天气预报、新闻报道、股票点评、各种黄页（如电话号码、航班和列车时刻表、地图等) 、免费电子信箱，以多种形式满足读者的需要。
　　5）向商业化方向发展
　　随着互联网上信息服务和电子商务的发展，用户基数的不断增长，为电子信息的增值服务提供了广阔的空间。在这里汇集了最新的思想、最先进的技术和最大的潜在市场。搜索引擎已经成为一项产业，它的商业利益成为推动系统完善和扩展的主要动力，网络信息的检索与利用由公用性转向商业化。
　　6）向专业化方向发展
　　由于用户千差万别，对信息搜索往往有不同的要求。综合性的搜索引擎收录的范围太广、太大、无法满足某一特定的需求。而垂直性专业搜索引擎则可解决这一难题。垂直性专业搜索引擎只面向某一特定的领域，专注于自己的特长和核心技术，能保证对该领域的信息的收录齐全与更新迅速。
　　7）向可视化方向发展
　　可视化信息检索是把文献信息、用户提问、各类检索模型以及利用检索模型进行信息检索的过程，展示在一个可视化空间中，并向用户提供信息检索服务。可视化搜索避免繁琐的操作过程,使用户更直接、方便、快速、高效地获取信息和知识。随着网络技术的不断发展，以及XML、RDF、GRID在信息中的应用，可视化检索将成为搜索引擎新的发展方向。
　　
　　
　　

3、算法原理、技术路线的分析和比较
3.1、搜索引擎原理
　　搜索引擎本质上是一个网络应用软件系统，它能够接受用户查询词或短语，然后根据一定的排序算法，在一定的时间内返回给用户检索结果。搜索引擎的工作流程一般可以分为网页搜集、网页预处理、检索服务三个部分。其中，网页搜集为网页预处理和检索提供信息来源；网页预处理根据相关策略整理网页信息，并建立索引，保障后续信息检索的有效进行；检索服务是最终面向用户的服务接口，也是搜索引擎的外在表现，它获取用户的检索词，返回给用户所需的检索信息。搜索引擎的上述工作流程如图 3.1 所示：
　　
　　①网页搜集器 
　　网页搜集器一般被称为网络机器人或者网络蜘蛛，这是由于其网页的整个抓
　　取过程类似于蜘蛛在蜘蛛网上爬行。一般采用的网页抓取策略为：将整个网络看做一个有向图，各个网页为该有向图中的节点，网页间的链接作为有向图的边。首先设置好起始种子 URL集 S，然后网络蜘蛛依次获取集合 S 中的URL，按照深度优先或广度优先策略抓取网页，存储到网页数据库中。同时解析该网页中的子链接 URL，如果子链接 URL 未被抓取过，则将其放入 S 中，并根据相关算法将 S中的 URL 按优先级排序。一般搜索引擎会同时启动多个网络蜘蛛，并行抓取网页资源。 
　　②网页预处理 
　　网页预处理作为搜索引擎工作流程中的中间环节，也是十分重要的一个环节，它主要包括页面解析、索引建立、网页重要度计算三个过程。 页面解析器通过对 html 文档进行分解，去除广告等内容无关信息，提取网页正文；运用切词工具，提取网页的关键词，去除停用词，最终形成网页的特征向量；提取并分析链接信息等。 索引器对网络蜘蛛抓取的网页建立高效索引，从而提高检索服务的效率。现在一般采用的索引数据结构是"倒排文件"，倒排文件是采用网页中的关键词作为索引，通过关键词索引关联网页。 
　　搜索引擎返回的检索页面是按页面重要度排序的，这样才能使用户在尽可能靠前的页面中找到所需的信息。与科技文献重要性评价方式一样，搜索引擎对网页的重要性评价一般也采用分析网页链接结构，计算引用次数来完成。在互联网上，存在着"中心网页"和"权威网页"，前者就像一个 Hub，其本身并没有什么主题内容，但拥有着许多连接到其他重要网页的链接；后者则被许多相关主题的网页所链接。可见，两者是一种相互补充的关系，一个中心网页必然有许多链接指向权威网页，而一个权威网页也必须被许多中心网页所链接。 
　　③检索服务 
　　检索服务是搜索引擎的外在表现，是与用户交互的接口。用户通过检索界面
　　输入检索词，通常是几个关键词句，通过分词器将关键词句划分成几个关键词，形成关键词集，各关键词显然都应该是包含在倒排文件词表中才是有意义的，这样每个关键词通过倒排文件索引获取网页集合，然后再取各关键词网页集合的交集，从而实现检索词与网页的匹配。最后再根据相关排序策略，将检索结果网页按优先级降序返回给用户。 
3.2、个性化信息检索
3.2.1、个性化信息检索概述
　　信息检索技术满足了人们一定的需要，但由于其通用的性质，仍不能满足不同背景、不同目的和不同时期的查询请求。个性化服务技术就是针对这个问题提
出的，它为不同用户提供不同的服务，以满足不同的需求。个性化服务通过收集
和分析用户信息来学习用户的兴趣和行为，从而实现主动推荐的目的。个性化服
务技术能充分提高站点的服务质量和访问效率，从而吸引更多的访问者。
为了实现个性化服务，首先需要跟踪和学习用户的兴趣和行为。
 3.2.2、用户描述文件
　　对个性化服务系统来说，最重要的是用户的参与。为了跟踪用户的兴趣与行为，有必要为每个用户建立一个用户描述文件（user profile）。用户描述文件刻画用户的特征以及用户之间的关系。在制定用户描述文件之前，需要考虑下面几个问题：
　　①收集什么数据？收集的数据用于什么目的？②如何收集数据？③收集的数据如何组织？④用户信息能否自适应地更新？
　　 在收集用户的信息之前，首先需要分析用户愿意提供什么信息，用户一般都很关心个人信息的保密性，www.cyberdialogue.com  的调查显示，80％用户愿意向 Web 站点提供自己的姓名、性别、年龄、教育背景和兴趣，但大多数用户不愿意提供私有、敏感的信息，如个人收入和信用卡号等。该公司另一项调查显示，28％的用户愿意 Web 站点向其他 Web 站点共享自己的信息。为了规范 Web用户信息的保密性，W3C 成立了 P3P 工作组来解决这个问题，它允许用户有选择地向 Web 站点提供自己的信息，从而达到保护用户信息的目的，目前已有一些 站 点 和 浏 览 器 支 持 了 P3P ， 如 www.w3c.rog  、 www.microsoft.com、www.aol.com、www.att.com  等站点和 Microsoft/AT&T 的还处于试用阶段的P3P 浏览器等等。

3.2.3、资源描述文件 
　　目前，个性化服务系统所处理的资源都属于文本范畴，资源的描述与用户的描述密切相关，一般的做法是用同样的机制来表达用户和资源。资源描述文件可以用基于内容的方法和基于分类的方法来表示。
　　基于内容的方法是从资源本身抽取信息来表示资源，使用最广泛的方法是用加权关键词矢量。对文档来说，关键的问题是特征选取，特征选取要达到两个目标：一是选最好的词；二是选取的词最少。完成文档特征的选取后，还得计算每个特征的权值，使用最广泛的是 TFIDF方法。
　　基于分类的方法是利用类别来表示资源，对文档资源进行分类有利于将文档推荐给该类文档感兴趣的用户。文本分类方法有多种，例如：朴素贝叶斯（Native-Bayes）、k 最近邻方法（k NN）和支持向量机（SVM）等。
　　资源的类别可以预先定义，也可以利用聚类技术自动产生。许多研究表明：
聚类的精度非常依赖于文档的数量，而且由自动聚类产生的类型可能对用户来说是毫无意义的，因此可以先使用手工选定的类型来分类文档，在没有对应的候选类型或需要进一步划分某类型时，才使用聚类产生的类型。
3.2.4、个性化推荐
　　个性化推荐可以采用基于规则的技术、基于内容过滤的技术和协作过滤技术。
a)基于规则的技术
　　基 于 规 则 的 系 统 如 IBM 的 Web Sphere(www.ibm.com/websphere) 、
Broad Vision(www.broadvision.com)、ILOG(www.ilog.com)等，其规则可以由用户定制，也可以利用基于关联规则的挖掘技术来发现。利用规则来推荐信息依赖于规则的质量和数量。基于规则的技术的缺点是随着规则的数量增多，系统将变得越来越难以管理。一个规则本质上是一个 IF-THEN 语句，规则可以利用用户静态属性来建立，也可以利用用户动态信息来建立。为了利用规则来推荐资源，用户描述文件和资源描述文件需要用相同的关键词集合来进行描述。信息推荐时的工作是这样的：首先根据当前用户阅读过的感兴趣的内容，通过规则推算出用户还没有阅读过的感兴趣的内容，然后根据规则的支持度（或重要程度），对这些内容排序并展现给用户。
 
　　基于规则的系统一般分为 3 部分(如图 3.2)：关键词层、描述层和用户接口层。关键词层提供上层描述所需的关键词，并定义关键词间的依赖关系，在该层可以定义静态属性的个性化规则；描述层定义用户描述和资源描述，由于描述层是针对具体的用户和资源，所以描述层的个性化规则是动态变化的；用户接口层提供个性化服务，根据下面两层定义的个性化规则将满足规则的资源推荐给用
户。
b)信息过滤技术
信息过滤技术可分为基于内容过滤的技术（content-based filtering）和协作过滤技术（collaborative filtering），基于内容过滤的系统如 Personal Web Watcher、Citeseer等，如图 2.3 所示，基于内容过滤的技术是通过比较资源与用户描述文件来推荐资源，它的关键问题是相似度计算，对于矢量空间模型来说，通常采用余弦相似度计算。如果用户的描述文件没有正确描述用户的兴趣和行为，那么该方法推荐的数据可能和用户真正的兴趣根本不相关。基于内容过滤的系统其优点是简单、有效。缺点是难以区分资源内容的品质和风格，而且不能为用户发现新的感兴趣的资源，只能发现和用户已有兴趣相似的资源。
　　协作过滤是根据用户的相似性来推荐资源。它与基于内容的过滤技术不同，
它比较的是用户描述文件，而不是资源与用户描述文件。其关键问题是用户聚类。
由于它是根据相似用户来推荐资源的，所以有可能为用户推荐出新的感兴趣的内
容。

　　基于近邻用户的协作过滤技术应用比较普遍，如 Anatagonomy，它的核心问题是为当前用户寻找 k 个最相似的邻居来预测当前用户的兴趣。该方法在实践
过程中遇到两个难题，一个是稀疏性，也就是指在系统使用初期，由于系统资源
还未获得足够多的评价，该方法很难利用这一些评价来发现相似的用户。另一个
是可扩展性，也就是指随着系统用户和资源的增多，该方法的性能会越来越低。
3.3、系统的框架
　　设计一个智能搜索引擎系统主要目的是针对中小型网站能进行自动信息检索建立排序文件,同时为用户提供快速信息查询服务,帮助用户检索到自己需要的知识。一个搜索引擎主要由网页搜索模块、文件索引模块、用户查询模块三大部分构成,这是一个搜索引擎必备的三部分功能。本文设计的搜索引擎扩展了这三大功能。
　　个智能搜索引擎系统需要良好的硬件支持和软件支持。针对第一章中所分析的现有搜索引擎存在的三大问题:如何建立一个通用的、有效的、广泛适用的模式问题;提高搜索引擎对自然语言的理解问题;搜索引擎的评价方法的改进问题。本文从硬件和软件两个方面进行了仔细地分析,设计了一个功能强大的搜索引擎系统,具体硬件框架和软件框架参见下面两节。
　　可移植性:采用Java技术具有良好的可移植性,Java语言是现在应用最广泛的开发语言,它可以在Windows,Linux等不同平台上运行,用Java语言开发本系统可以使本系统直接在不同的平台上应用。
　　可扩展性:采用分布式设计的方案。在硬件的设计上本论文采用了索引服务器和查询服务器分离开,这样可以保障系统良好的扩展性,当系统负载增加的时候可以扩展服务器来采用分布式的应用来解决上述问题。
　　可定制性:用户可以根据需要更改系统,使它适合自己需要的专业搜索引擎,用户可以通过增加和删减系统的功能。
　　智能性:采用中文自然语言处理的方法,本系统通过对用户输入的文字进行分析后确定检索策略。提供个性化的服务:本系统通过挖掘日志进行用户行为学习,进行个性化的服务,不同的用户对相同的输入可以检索到不同结果。

4、应用场景和应用水平的分析和比较
　　
　　搜索引擎是从Internet上查找、获得信息的一个重要手段,在各个领域都已得到广泛应用。然而,传统的搜索引擎无法满足人们对信息服务智能化和个性化的要求。
　　所以，近年来，WWW网络发展迅猛，如何提供对WWW网上信息的高效、智能的检索机制已经成为计算机网络领域内的一个研究热点。本文在总结现有的网络搜索引擎技术优缺点的基础上，针对中文搜索引擎所涉及的若干问题作了深入细致的研究探讨。 随着Internet上的站点和网页越来越多，搜索引擎的数据库也越来越大，用户查询同一个关键词所得到的结果也就越来越多，然而成千上万的结果对用户并没有什么实际意义，用户关心的是能否迅速在开头几十个结果中找到自己需要的信息。因此，如何将收集到的网页进行有规律的排序也是搜索引擎的任务之一。
　　随着工nternet的普及、人工智能技术的发展、人们信息需求的增加以及对信息服务要求的提高使得智能搜索引擎具有了可靠的支撑平台、广阔的发展空间和良好的市场前景。本文的研究成果不但具有一定的理论意义和技术价值,更具有一定的经济价值,它将有助于人们对生活信息、娱乐信息、科研信息、商业信息的检索和获取,能够起到方便人们的生活、增强科研能力、扩大商业机会,其应用推广将带来极大的社会经济效益。
　　
5. 其它研究和应用情况
　　　国外从事用户访问行为研究的学者也比较多, C . Holscher 和 G. Strube 将用户划分为二类: 专业人 士、新手( Experienced Users and Casual Users) ,并进行 调查,不同知识背景的人对搜索引擎性能和效率的 评价标准是不同的,也决定了他们的访问行为是有 所偏好的 。 根据这些不同类用户的访问行为考虑 设计不同类型的搜索引擎. 但是,这类研究存在的问 题在于,对单个用户所关心的信息了解不够. 　　
　　通过对个性化搜索访问行为的研究探讨智 能化搜索引擎. M . Perkowitz 提出了 PageGather 算 法 ,利用 Web 历史日志挖掘用户的存取模式,从 一些用户访问过的候选页面中归纳出一些优化的页 面访问序列,满足用户个性化服务的需要. R. Barrett 和P. Maglio 等人研制开发了WBI 系统 ,通过这些 Agent 在用户工作站上完成对用户最近访问的数据 流的观测、记录,并对过去发生的访问数据流进行更 新. 这些Agent 可以从记录的数据中获取用户的访 问行为. 国内对个性化的搜索引擎也已有一些研究, 张卫 丰 等 人 提 出 了 个 性 化 代 理 系 统 的 结 构 ( PSA) ,韩立新等人利用用户的访问模式、类层次 结构和多关键词构造个性化信息检索系统等等 . 信息过滤机制也属于个性化检索的研究范畴 ,信 息过滤系统的主要目标是匹配信息空间和用户兴趣 空间. 现在的个性化检索的研究存在不足,表现在: 1)不能综合考虑用户的浏览行为与用户检索页面的 内容; 2)对用户多方面兴趣的考虑仍不够; 3)个性化 检索的自动化程度不高,需要利用机器学习技术对 用户提供大量的反馈信息进行训练; 4)不能准确把 握用户的信息需求且不能很好地适应用户需求的变 化. 　　	
　　Internet 上分布式智能化信息获取方法的研 究. 研究人员引入分布式系统和 Agent 的概念,针对 普遍存在的可扩展性、可靠性、可获得性和查询服务 智能性关键技术展开研究,试图构造面向 Internet 的 智能化分布式信息检索系统 . 　　
　　基于 KDD方法的智能化信息获取方法的研 究. 近几年,KDD( Knowledge Discovery in Database)已 成为一个十分活跃的领域,研究人员将 KDD、DM 与 网上搜索引擎的研究结合起来,用以研制高质量、高 效率的检索工具 . 目前的研究成果还存在两 点不足: 1)主要表现在发现的规则的可用性还不理想(模式和规则中领域知识、信息不够); 2)算法对数 据过于敏感而导致模式粒度过小而数量过大. 对于 搜索引擎,一般采用速度、查准率、查全率和失效率 来评价其性能。
　　
　　
　　
　　
　　
　　
6、总结
　　借助人工智能、KDD 方法以及其他数学工具探 索一种高效、智能化的信息获取工具,获得较快的查 询速度、较高的查准率及查全率,能够为用户提供方 便、安全、快捷、智能和个性化的信息服务. 下面列出 了若干在信息获取技术研究方面值得研究讨论的问 题及相应的关键技术: 　　
　　通过对用户的查询计划、意图、兴趣方向进 行推理、预测和分类,研究新的智能化搜索算法 　　
　　a 研究和讨论智能搜索引擎问题,提高检索的 查全率和查准率,这可以看作一个优化问题. 通过对 用户最近访问页面内容的分析,更新维护用户兴趣 词典( profiles). 比如,可以用到人工智能的一些工 具,如遗传算法、神经网络、机器学习方法,寻找到一 组最优的能够代表用户兴趣的查询条件. 　　
　　b 研究和讨论新的复杂度小、高效动态增量式 的聚类算法. 通过对用户访问日志中数据的清洗(消除噪音) ,获得用户访问的路径信息,挖掘出用户访 问的兴趣知识,以此调整文档的索引结构,改善搜索 引擎的性能,以获得更好的检索效率. 　　、
　　c 结合决策分析中的评价方法,提出新的网页 评价方法,提高检索效率. 　　	d 借鉴Agent 的思想研究和探讨应用于分布式 智能化搜索引擎模型 MAS 的组织方式、思维方式、 行为方式. 这一部分研究的关键技术是: 　　
　　①智能化搜索引擎的模型描述. 可以借鉴 Agent 的组织方式、思维方式、行为方式构造分布式智 能化搜索引擎模型. 　　
　　②研究构造快速、高效、低复杂度的动态增量 聚类算法和并行算法来获知用户访问信息,以提高 访问的速度和检索效果. 　　
　　利用本体( Ontology)良好的概念层次、对逻 辑理解的支持研究智能化的搜索引擎目前的搜索引擎技术处于相当原始的阶段,提 供的是相当原始的查询操作,在查询时需要提供较 为详细的语义说明才能获得较为相关的数据. 提出了 Web 数据的语义查询问题,利用领域知识作为上下文关系本体去规范 Web 的 查询操作.试图描述并产生一种计算机可 以理解的知识库( Knowledge Base) ,这种知识库代表 人的见解,可以被看为一种本体,能使检索更加有 效.在信息论中,信息是可以进行度量的,可以根据 其负熵的大小进行客观的评价. 可以考虑将信息负 熵表达方法与用户的知识本体相结合, 以实现个性 化的信息检索服务. 　　
　　讨论查全率、查准率与页面信息的可靠性 关系在搜索引擎中可靠性的因素并不是可以忽略不 计,反而它对搜索的最终结果将产生较大的影响(受 到页面错误、搜索引擎中的人为错误、本体差异、不 对称知识等因素的影响) ,在这种情形下研究讨论查 全率、查准率与页面信息的可靠性关系有一定的意义.
